# Contributing to Criminal Appeal

Thank you for considering contributing to the Criminal Appeal project! We welcome contributions from everyone.

## Bug Reports
If you find a bug, please open an issue in the repository and provide the following information:
- A clear description of the bug
- Steps to reproduce the bug
- Expected and actual behavior
- Screenshots or logs, if applicable

## Feature Requests
We are open to feature requests! Please consider the following when submitting a request:
- Explain the feature clearly and why it's valuable
- Provide any examples or use cases

## Pull Requests
To contribute code, follow these steps:
1. Fork the repository and clone it to your local machine.
2. Create a new branch for your feature or bug fix.
3. Write your code and ensure it follows our coding standards.
4. Test your changes thoroughly.
5. Submit a pull request with a clear description of your changes.

## Code of Conduct
Please read our [Code of Conduct](CODE_OF_CONDUCT.md) to ensure a welcoming environment for all contributors.

## Development Setup
To set up the development environment, follow these steps:
1. Clone the repository.
2. Install the required dependencies using `npm install`.
3. Follow additional setup instructions in the documentation.

## Testing
We have a set of unit and integration tests. To run them:
- Use `npm test` to run all tests.
- Ensure that all tests are passing before submitting a pull request.

## Commit Message Conventions
Please follow these guidelines for commit messages:
- Use the present tense: "Add feature" not "Added feature".
- Start with a capital letter.
- Limit the first line to 72 characters or less.
- Include a detailed description if necessary.

Thank you for your contributions!
